# TaxiSH: Large-Scale Taxi Demand and Mobility Flow Dataset in Shanghai, China

[![Python 3.8+](https://img.shields.io/badge/python-3.8+-blue.svg)](https://www.python.org/downloads/)

This repository contains the official data processing and validation pipeline for the **TaxiSH Dataset** (Large-Scale Taxi Demand and Mobility Flow Dataset in Shanghai, China). 

The codebase transforms 133 GB of raw, noisy GPS streams (over 2.16 billion records) into high-fidelity, privacy-preserving, and topologically validated urban mobility datasets (Trip-Level OD, OD Flow, and In/Out Flow).

## 🗂️ Repository Structure

```text
TaxiSH/
│
├── code/
│   ├── stage1_get_occupied_trip.sh          # Raw stream segmentation based on taximeter status
│   ├── stage2_point_level_refinement.py     # Heuristic intra-trip noise filtering (drifts, angles)
│   ├── stage3_quality_assurance.py          # Trip-level QA (spatial-temporal continuity constraints)
│   ├── stage4_get_trip_od.sh                # OD extraction and distance/duration calculation
│   ├── stage5_spatil_kinematic_filter.py    # Spatial joining (TAZ) and global kinematic bounding
│   ├── stage6_dataset_generate.py           # Generation of the 3 final CSV datasets
│   └── stage7_generate_geometry.py          # WKT geometry generation for standard Grid IDs
│
│
└── README.md

```

## 💻 System Requirements & Dependencies

To handle the massive scale of the original dataset, the pipeline utilizes multiprocessing and optimized vectorized operations.

**Recommended Hardware:**

* RAM: 64GB+ (for full dataset processing)
* OS: Linux (Ubuntu/CentOS recommended for `bash` and `awk` compatibility)

**Python Dependencies:**

```bash
pip install pandas numpy geopandas scipy tqdm

```

## 🚀 Execution Order (The Pipeline)

To reproduce the data processing or apply this methodology to your own trajectory data, please execute the scripts in the following strict order:

### Micro-level Refinement

* **Stage 1 & 2:** Extract candidate trips from raw logs and remove intra-trip noise (stationary points, acute angles, and velocity anomalies).
* **Stage 3:** Filter out structurally corrupted trajectories (e.g., GPS teleportation, long signal loss).

### Macro-level Validation & Aggregation

* **Stage 4 & 5:** For processing large batches, it is recommended to run the metric calculation and spatial joining sequentially in the background using `nohup`:
```bash
nohup bash -c "bash stage4_get_trip_od.sh && python -u stage5_spatil_kinematic_filter.py" > pipeline.log 2>&1 &

```


* **Stage 6:** Run `python stage6_dataset_generate.py` to aggregate the validated records into three standardized products: `TaxiSH_Trip_Level_OD.csv`, `TaxiSH_OD_Flow.csv`, and `TaxiSH_InOut_Flow.csv`.
* **Stage 7 (Optional):** Run `python stage7_generate_geometry.py` to generate the `Shanghai_Grid_Geometry.csv` file for immediate GIS mapping and visualization.

## 🧪 Toy Sample Testing

If you wish to test the pipeline without downloading the entire 133GB raw dataset, we have provided a miniature dataset in the `sample_data/` directory.

1. Navigate to the `code/` directory.
2. Modify the `input_file` paths in the scripts to point to `../sample_data/raw_sample.txt`.
3. Run the scripts sequentially. The entire toy pipeline should execute in under 10 seconds.
